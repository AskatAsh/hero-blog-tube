The scope of var, let, and const

In javascript, Var, Let and Const are keywords that are used to declare variables. Var was used until ES6. After ES6, two new variables Let and const are introduced.

The scope of var, let and const are discussed here:

1. Var

  • Function Scope: Variables declared with var are function-scoped, meaning they are only accessible within the function in which they are declared.
  • Block Scope: var does not have block-level scope, so if a var variable is declared inside a block (like an if statement or a loop), it is still accessible outside that block.

2. Let

  • Function Scope: Variables declared with let are also function-scoped. They are not be accessible outside that function.
  • Block Scope: Variables declared with let have block-level scope. They are accessible only within the block (like if statement, loop) in which they are defined.

3. Const

  • Constant Value: const is used to declare constants, and the value of a const variable cannot be reassigned after it's initialized. However, the contents of an object or array declared with const can still be modified, as long as the variable itself is not reassigned.
  • Block Scope: Like let, variables declared with const are block-scoped.

In modern javascript, Var is not used anymore. It's recommended to use let and const because they are better at scoping and they also help in writing bug free code.


There is another reason for using let and const over var and it is because of hoisting.


Variables declared with var are hoisted to the top of their containing function or global scope. This means you can access them before their declaration, but they will be undefined until they are assigned a value.


Variables declared with let are also hoisted, but unlike var, they are not initialized until their declaration is reached, so you cannot access them before the line where they are declared.

The use cases of null and undefined

In many programming languages, including JavaScript, null and undefined are two distinct values with specific use cases.

The use cases of null and undefined discussed here:

1. Null

  • Explicit Absence of Value: Null is often used to explicitly indicate the absence of a value. It is a way of saying that a variable or object property has no meaningful value or hasn't been assigned a value yet.
  • Database and API Responses: Null is commonly used in database systems and API responses to indicate that a field or property has no data associated with it.

1. Undefined

  • Variable Declaration: In JavaScript, a variable that has been declared but hasn't been assigned a value is undefined.
  • Function parameters and return value: Function parameters that are not provided with values become undefined.
    If a function does not have a return statement or explicitly returns a value, it will return undefined.

What do you mean by REST API?

The full meaning of RESTful API (REST API) is Representational State Transfer. It's a type of web API that is designed to enable the communication and interaction between different software systems over the internet.

Here are some key points REST API:

  • Uniform Interface: REST APIs have a consistent and uniform set of operations, such as using HTTP methods like GET (for retrieving data), POST (for creating data), PUT (for updating data), and DELETE (for deleting data).
  • Resource-Based: Resources are the key abstractions in REST, and each resource is identified by a unique URI (Uniform Resource Identifier). Clients interact with these resources using a small set of standard methods.
